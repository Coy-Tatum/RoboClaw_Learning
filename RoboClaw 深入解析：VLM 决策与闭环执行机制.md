# RoboClaw 深入解析：VLM 决策与闭环执行机制

在 RoboClaw 框架中，视觉语言模型（VLM）并不直接输出底层物理参数，而是充当“指挥官”，通过动态调用底层技能代码来实现复杂的长周期任务。以下是其核心运行机制的拆解：

## 一、 VLM 工具调用流 (Tool Calling)

整个动作规划可以拆解为四个标准步骤：

1. **定义技能库 (Tool Registry)**：将机器人底层的物理控制动作（如 `skill_pick_up`）封装为 Python 函数，并配置结构化的“说明书”（JSON Schema），明确技能名称和所需参数。
2. **构建上下文 (Context)**：主程序在执行前，将系统提示词、可用工具列表、历史记录以及**当前摄像头的实时切片（快照）**打包发送给 VLM。
3. **推理与结构化输出**：VLM 基于画面和任务进行推理，输出包含具体决策的 JSON 数据（例如指定工具名和目标物体）。
4. **动态解析与执行**：主程序解析 JSON，通过工具注册表（字典）动态匹配并运行对应的底层 Python 函数，驱动机械臂。

---

## 二、 视觉切片与执行评估 (Execution Monitoring)

机器人的闭环控制高度依赖于关键节点的“视觉切片”（Camera Slicing）。

* **单次动作时序**：执行正向动作 ➡️ 动作结束保持状态 ➡️ 摄像头切片 ➡️ VLM 评估。
* **校验提示词 (Evaluation Prompt)**：主程序会将包含上一步动作参数的模板化提示词（例如：“你刚才尝试执行了[抓取]，参数是[水杯]，请判断是否成功？”）连同最新切片发送给 VLM。
* **按需触发 EAP**：只有当 VLM 判断动作**失败**（或处于纯数据收集模式）时，系统才会触发“纠缠动作对（EAP）”的反向动作，让机械臂退回初始安全位置。

---

## 三、 故障恢复与状态机 (Failure Recovery)

当动作执行失败并触发 EAP 退回原位后，系统通过以下机制避免陷入死循环：

1. **让出视野**：机械臂退回安全位置（Home Pose），确保全局摄像头能拍到无遮挡的“案发现场”。
2. **构建重试上下文**：主程序会将**“上一次失败的操作参数”**与**“退回原位后拍下的最新切片”**同时喂给 VLM。这使得 VLM 能够“长记性”，并基于物体的新位置（即使物品已经倾倒或滚落）重新规划动作路径。
3. **人工升级 (兜底机制)**：如果目标物体掉出视野导致 VLM 无法规划，或重试达到上限，状态机会停止物理动作，并通过预设接口（如飞书）将报错文本与现场照片推送给人类工程师，请求场外援助。
